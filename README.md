# NLP Recommendation Chatbot  
研究分野の興味を対話的に推定し、最適な研究室を推薦するチャットボット

## 1. 概要
本プロジェクトは、  
**「曖昧な興味表明からでも、対話を通じて関心分野を明確化できる推薦システム」**  
の構築を目的として開発した、NLPベースの対話型研究室推薦チャットボットです。

ユーザの自由入力文を解析し、研究分野ごとのスコアリングを行うとともに、  
スコアが拮抗した場合には **逆質問（フォローアップ質問）を行う対話設計** を導入しています。

単発の推薦にとどまらず、  
ユーザの思考を整理しながら推薦精度を段階的に高めることを重視した設計としています。

---

## 2. 背景・課題設定
研究分野や研究室の選択において、ユーザは以下のような課題を抱えがちです。

- 興味はあるものの、言語化が難しい  
- 初期入力のみでは本質的な関心を把握しにくい  
- 一度の推薦で結論を出すことに不安を感じる  

そこで本プロジェクトでは、

- 入力文に基づく **軽量な興味推定**  
- 対話を通じた **段階的な嗜好の精緻化**

を組み合わせることで、ユーザの負担を抑えつつ、  
納得感のある推薦を実現する設計を採用しました。

---

## 3. 主な機能と設計意図

### 3.1 名詞ベースの興味推定（NLP）
- Janome を用いた形態素解析を実施  
- 自由入力文から **名詞のみを抽出**  
- 研究室ごとに定義した研究キーワード辞書と照合  

| 判定方法 | スコア |
|----------|--------|
| 完全一致 | 3点 |
| 部分一致 | 1点 |

**設計意図**  
本実装では高度なモデルは用いず、ルールベース方式を採用しています。  
これにより、スコアの重みづけの調整を手動で行いやすい構成となっています。

---

### 3.2 研究室単位のスコアリング制御
- 各研究室ごとにスコアを保持  
- 1回の入力につき、同一研究室では最大スコアのみを反映  

**設計意図**  
ノイズとなる単語の多重加算を防ぎ、  
「強い関心が一度示された」事実を適切に評価することを目的としています。

---

### 3.3 逆質問（フォローアップ質問）による精緻化
スコアが拮抗している場合、一定確率で逆質問を実施します。

- 2択の研究分野候補を提示  
- 選択結果に応じて関連研究室のスコアを加算  
- 「どちらも興味なし」が選択された場合は減点  

**設計意図**  
ユーザに長文入力を求めることなく、  
最小限の操作で関心分野を収束させることを意図しています。

---

### 3.4 推薦判定ロジック
- スコア上位1位と2位の差が一定以上となった場合、推薦を確定  
- 差が小さい場合は対話を継続  

途中で `列挙` と入力することで、  
現在のスコア上位3研究室を確認できます。

---

## 4. 対話フロー
1. ユーザが興味分野を自由入力  
2. 名詞抽出およびスコア更新  
3. 必要に応じて逆質問を提示  
4. スコア差が十分となった時点で研究室を推薦  

---

## 5. 技術スタック
- Python  
- Janome（形態素解析）  
- ルールベース推薦ロジック  
- ChatServer（Webチャットサーバ）
